## FASE 2: ANÁLISIS ESTADÍSTICO (Z-SCORE)
══════════════════════════════════════════

Con los 100 precios de cierre recolectados, calcula:

### 2.1 Media Móvil Simple (SMA100)
```
SMA = (Σ prices[i] para i=0..99) / 100
```

### 2.2 Desviación Estándar de la Muestra
```
Varianza = Σ((prices[i] - SMA)^2) / (N - 1)
σ = sqrt(Varianza)
```

### 2.3 Z-Score del precio actual
```
Z = (precio_actual - SMA) / σ
```

### 2.4 Percentil de la distribución normal
```
Z < -2.0  →  < 2.3% de los precios históricos  →  SOBREVENTA EXTREMA
Z > +2.0  →  > 97.7% de los precios históricos →  SOBRECOMPRA EXTREMA
-1 < Z < +1 → Rango normal (±68%)               →  ZONA MUERTA
```

### 2.5 Validación de señal
Antes de generar señal, verifica:
- [ ] σ > 0 (hay volatilidad)
- [ ] Volumen de las últimas 5 velas > min_volume_sol
- [ ] No hay posición abierta del mismo tipo
- [ ] daily_pnl_sol > -0.05
- [ ] No estamos en cooldown post-stop-loss

## FASE 3: TOMA DE DECISIÓN
════════════════════════════

```
SI z_score < -2.0 AND validation_passed AND NOT open_position_long:
    → SEÑAL: BUY
    → ACCIÓN: Swap USDC → SOL via Jupiter
    → CANTIDAD: min(max_trade_sol, wallet_balance_sol * 0.4)

SI z_score > +2.0 AND validation_passed AND open_position_long:
    → SEÑAL: SELL
    → ACCIÓN: Swap SOL → USDC via Jupiter
    → CALCULAR: profit = (precio_venta - precio_compra) * cantidad_sol

SI open_position_long AND z_score < -3.5:
    → SEÑAL: STOP_LOSS
    → ACCIÓN: Cerrar posición inmediatamente

EN CUALQUIER OTRO CASO:
    → SEÑAL: HOLD
    → ACCIÓN: Registrar Z-Score en log
```

### Kelly Criterion (tamaño de posición):
```
win_rate   = trades_ganadores / total_trades  (mínimo 10 trades para usar)
kelly_pct  = win_rate - (1 - win_rate) / razon_riesgo_recompensa
trade_size = min(kelly_pct * 0.5, max_trade_sol)

Si <10 trades históricos: usar max_trade_sol * 0.5
```
