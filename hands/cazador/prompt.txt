## FASE 1: ANÁLISIS FRACTAL Y DE CRITICALIDAD

Obtén las últimas 200 velas horarias de SOL/USDC.
Ejecuta el motor matemático:

```bash
python analysis/fractal_engine.py '{"prices": [LISTA_DE_200_PRECIOS_CIERRE]}'
```

El script devuelve JSON con:
  - hurst: Exponente de Hurst actual
  - kurtosis: Curtosis de los retornos
  - is_critical: booleano — ¿caída brusca del Hurst?
  - sentiment: clasificación del Fear & Greed Index
  - sentiment_score: valor 0-100
  - confidence_score: 0.0 - 1.0
  - signal: "WAIT" | "ACTIVATE" | "ZERO_CLAW_ATTACK"

**Interpretación:**
  signal = "WAIT"             → HOLD. Registrar métricas y terminar ciclo.
  signal = "ACTIVATE"         → Alerta amarilla. Evaluar compra de opción pequeña.
  signal = "ZERO_CLAW_ATTACK" → EJECUTAR. Máxima convicción. Sin dudas.

## FASE 2: VERIFICAR EL COLCHÓN

Consulta el balance de {RISK_HUNTER_WALLET} vía RPC.

Reglas de capital:
  balance < 0.10 SOL → No operar. Registrar "COLCHÓN INSUFICIENTE". Terminar ciclo.
  0.10 - 0.30 SOL   → Capital reducido: máximo 1 posición, 15% del colchón
  0.30 - 0.60 SOL   → Capital normal: máximo 2 posiciones, 15-20% del colchón
  > 0.60 SOL        → Capital completo: hasta 3 posiciones según señal

Verificar posiciones abiertas (DB: open_positions WHERE hand_id='cazador'):
  Si ya hay 3 posiciones activas → No abrir nuevas. Gestionar las existentes.

## FASE 3: SELECCIÓN Y EJECUCIÓN DE OPCIONES OTM

**Solo ejecutar si signal ≠ "WAIT" y hay colchón suficiente.**

### 3.1 Determinar tipo de opción:
```
sentiment_score < 30 → Comprar PUT  (pánico → caída probable)
sentiment_score > 70 → Comprar CALL (euforia → subida explosiva posible)
30 ≤ score ≤ 70      → Comprar STRADDLE (call + put simultáneos)
```

### 3.2 Calcular strike objetivo:
```
precio_actual = último precio de cierre
CALL: strike = precio_actual × 1.15  (15% fuera del dinero)
PUT:  strike = precio_actual × 0.85  (15% fuera del dinero)
```

### 3.3 Ejecutar compra de opciones via script:
```bash
# Para ACTIVATE (señal amarilla):
python scripts/zeta_options.py \
  --action buy_option \
  --type {call|put|straddle} \
  --strike {strike_calculado} \
  --expiry 14 \
  --max_premium_sol 0.03 \
  --max_contracts 5 \
  --signal ACTIVATE

# Para ZERO_CLAW_ATTACK (máxima convicción):
python scripts/zeta_options.py \
  --action buy_option \
  --type {call|put|straddle} \
  --strike {strike_calculado} \
  --expiry 7 \
  --max_premium_sol 0.05 \
  --max_contracts 10 \
  --signal ZERO_CLAW_ATTACK
```

## FASE 4: GESTIÓN DE POSICIONES ACTIVAS

Para cada posición abierta en la DB (hand_id='cazador', status='open'):

```bash
# Consultar precio actual de la opción:
python scripts/zeta_options.py --action check_price --position_id {id}
```

Reglas de cierre:
  precio_actual > precio_compra × 5.0  → TAKE PROFIT (realizamos 5x mínimo)
  precio_actual < precio_compra × 0.20 → STOP LOSS (80% pérdida — salir limpio)
  tiempo_restante < 24h AND profit < 2x → CERRAR (theta nos destruye)

Si se cierra con GANANCIA > 10x → activar Protocolo de Inyección de Capital:
```bash
python scripts/zeta_options.py --action distribute_profit \
  --profit_sol {ganancia_total} \
  --profit_wallet {PROFIT_WALLET} \
  --main_wallet {MAIN_WALLET} \
  --multiplicador_wallet {MULTIPLICADOR_WALLET}
# Distribución: 50% PROFIT_WALLET / 25% Bot1 / 15% Bot2 / 10% reserva
```
